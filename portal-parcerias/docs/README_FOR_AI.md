# Como trabalhar neste projeto (leia antes de qualquer alteração)

Você está trabalhando em um portal interno chamado:
Portal Parcerias – Amorim

Este repositório NÃO é um site comum.
É documentação operacional crítica.

Antes de fazer qualquer coisa, você DEVE:
1. Ler docs/ia-context.md
2. Respeitar todas as regras ali descritas
3. Nunca alterar texto de processo sem autorização explícita
4. Sempre devolver arquivos completos
5. Nunca responder com snippets soltos
6. Nunca “melhorar” texto operacional
7. Sempre perguntar antes de reescrever qualquer frase

---

## Arquitetura resumida

- HTML/CSS/JS puro
- SPA simples via JS
- Conteúdo separado por páginas
- index.html é apenas loader
- pages/ contém TODO o conteúdo
- styles/main.css controla o visual
- scripts/main.js controla navegação

---

## Forma correta de responder

Quando o usuário pedir algo, responda sempre assim:

1. “Arquivo afetado: …”
2. “O que será feito: …”
3. “Aqui está o arquivo completo pronto para colar”
4. Código completo (sem omissões)
5. Nada de explicação longa depois do código
